## Tech Debt (3 items, all Phase 4 workflow template)

### Phase 04: OAuth Consent Onboarding

**Item A — Token exchange error handling (Medium)**
- `workflows/fortnox-consent-onboarding.json`, node "Exchange Code for Token"
- Uses `continueRegularOutput` — HTTP errors from Fortnox token endpoint don't route directly to error page
- Currently mitigated by downstream "Decode JWT" node throwing on missing access_token
- Fix: Change to `continueErrorOutput` with explicit connection to "Show Error Page"
- Affected requirements: AUTH-05, AUTH-07

**Item B — Company info verification failure (Medium)**
- Same file, node "Verify Consent - Get Company Info"
- Uses `continueRegularOutput` — API failure silently stores `companyName='Unknown Company'`
- Consent is recorded even when verification call fails
- Fix: Change to `continueErrorOutput` or detect error payload in "Prepare Consent Record"
- Affected requirements: AUTH-07, COMP-01

**Item C — CSRF state token not deleted after use (Low)**
- Same file, no "Delete State Token" node after validation
- State token remains in Data Table, allowing replay within 10-minute window
- Fix: Insert dataTable delete node between "Verify State Token" and "Build Auth Header"
- Affected requirements: AUTH-06

**Total:** 3 items across 1 phase. All are workflow template error handling improvements — none block core node functionality.
